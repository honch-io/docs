The Arduino ESP32 SDK is a C++ wrapper around Shared Core. It targets ESP32 boards using the Arduino framework and exposes a singleton-style `Honch` API.

Status: in progress, library version `0.1.0`.

## Before You Start

You need:

| Requirement | Notes |
| --- | --- |
| Board family | ESP32 Arduino only. |
| Arduino core | ESP32 Arduino core with `WiFi`, `Preferences`, and `HTTPClient`. |
| Network | Wi-Fi connection before flush. |
| Project key | Sent as `X-Honch-Project-Key`. |
| Event buffer | Caller-owned RAM buffer. |

Not included: relay, OTA, and non-ESP32 Arduino boards.

## 1. Add The Library

Include the library in an ESP32 Arduino project that can compile the SDK sources:

```cpp
#include <WiFi.h>
#include <Honch.h>
```

Expected result: your sketch can reference `HonchConfig` and the singleton `Honch` object.

## 2. Configure Wi-Fi First

The Arduino wrapper does not own Wi-Fi setup. Connect before calling `Honch.flush()`:

```cpp
WiFi.begin("ssid", "password");
while (WiFi.status() != WL_CONNECTED) {
  delay(250);
}
```

Expected result: `WiFi.status()` returns `WL_CONNECTED`.

## 3. Configure Honch

```cpp
struct HonchConfig {
  const char *apiKey;
  const char *host;
  const char *deviceModel;
  const char *firmwareVersion;
  const char *environment;
  uint8_t *eventBuffer;
  size_t eventBufferSize;
  uint32_t flushIntervalSeconds;
  uint32_t flushEventThreshold;
  bool insecureSkipTlsVerify;
};
```

Required fields:

| Field | Example |
| --- | --- |
| `apiKey` | `project-key` |
| `host` | `https://i.honch.io` |
| `deviceModel` | `esp32-devkit` |
| `firmwareVersion` | `1.0.0` |
| `eventBuffer` | Static byte buffer. |
| `eventBufferSize` | Buffer size in bytes. |

`eventBufferSize` is passed to the core as `max_event_bytes`. Queue state is stored through ESP32 `Preferences`.

## 4. Send A First Event

```cpp
#include <WiFi.h>
#include <Honch.h>

static const char *WIFI_SSID = "ssid";
static const char *WIFI_PASSWORD = "password";
static const char *HONCH_PROJECT_KEY = "project-key";

static uint8_t eventBuffer[8192];

void setup() {
  Serial.begin(115200);

  WiFi.begin(WIFI_SSID, WIFI_PASSWORD);
  while (WiFi.status() != WL_CONNECTED) {
    delay(250);
  }

  HonchConfig config = {};
  config.apiKey = HONCH_PROJECT_KEY;
  config.host = "https://i.honch.io";
  config.deviceModel = "esp32-devkit";
  config.firmwareVersion = "1.0.0";
  config.environment = "production";
  config.eventBuffer = eventBuffer;
  config.eventBufferSize = sizeof(eventBuffer);
  config.flushIntervalSeconds = 60;
  config.flushEventThreshold = 30;
  config.insecureSkipTlsVerify = false;

  if (Honch.begin(config)) {
    Honch.track("app_started", "{\"source\":\"arduino\"}");
    Honch.flush();
  } else {
    Serial.println(Honch.lastError());
  }
}

void loop() {}
```

Expected result: `Honch.begin(config)` returns `true`, `app_started` is queued, and `Honch.flush()` attempts delivery to `<host>/capture`.

## 5. Use The Public API

```cpp
bool Honch.begin(const HonchConfig &config);
bool Honch.track(const char *eventName, const char *propertiesJson = "{}");
bool Honch.identify(const char *distinctId, const char *traitsJson = "{}");
bool Honch.setProperty(const char *key, const char *valueJson);
bool Honch.sessionStart(const char *sessionName);
bool Honch.sessionEnd();
bool Honch.flush();
bool Honch.shutdown();
bool Honch.reset();
const char *Honch.deviceId();
const char *Honch.lastError();
```

The wrapper returns `bool` and stores the last SDK status. Call `Honch.lastError()` immediately after a failed call.

## 6. Add Identity And Sessions

```cpp
Honch.identify("account-123", "{\"plan\":\"beta\"}");

Honch.sessionStart("recording");
Honch.track("recording_started", "{\"mode\":\"hdr\"}");
Honch.sessionEnd();
```

Expected result: identified events use the configured distinct ID, and session events add `$session_id`.

## Runtime Behavior

- SDK platform is `arduino-esp32`.
- Device ID is derived from the ESP32 eFuse MAC as `esp32-<12 uppercase hex chars>` when running on Arduino, then passed to the core as configured identity.
- Background flush is disabled in the current wrapper; call `Honch.flush()` explicitly.
- Queue capacity is 256 entries in the Arduino `Preferences` adapter.
- `Honch.track`, `Honch.identify`, and `Honch.setProperty` use JSON strings.
- `Honch.shutdown()` queues `$device_shutdown`, attempts a final flush, and releases the core client.

## Transport

The transport appends `/capture` to `host` and sends:

```text
Content-Type: application/vnd.honch.chunk
X-Honch-Project-Key: <apiKey>
X-Honch-Stream-Id: <stream_id>
```

HTTPS uses `WiFiClientSecure`. Set `insecureSkipTlsVerify` only for local testing where certificate verification is intentionally disabled.

## Examples

- `examples/HonchBasic/HonchBasic.ino` connects to Wi-Fi, initializes Honch, tracks `boot`, and flushes.
- `examples/HonchOfflineQueue/HonchOfflineQueue.ino` queues before Wi-Fi, then connects and flushes.

If you use Arduino CLI, you can compile the examples from the SDK repository:

```bash
arduino-cli compile --fqbn esp32:esp32:esp32 ports/arduino/examples/HonchBasic
arduino-cli compile --fqbn esp32:esp32:esp32 ports/arduino/examples/HonchOfflineQueue
```

## Troubleshooting

| Symptom | Check |
| --- | --- |
| `Honch.begin` returns `false` | Required config fields and event buffer. |
| `Honch.flush` returns `false` | Wi-Fi, host, TLS, project key, or retryable response. |
| No delivery after tracking | Background flush is disabled; call `Honch.flush()`. |
| TLS fails locally | Use a proper HTTPS endpoint or intentionally set `insecureSkipTlsVerify` for local tests. |
| Queue fills | Network or Capture response is retryable; reduce event volume or fix delivery. |
