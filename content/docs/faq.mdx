title: FAQ
description: Short answers for common Honch SDK integration questions.
---

## Which SDK should I use?

Use ESP-IDF for ESP32 firmware built with ESP-IDF, C/POSIX for embedded Linux or local validation, MicroPython for firmware that can include `_honch_core`, Arduino ESP32 for ESP32 sketches, and React Native Relay for companion apps that forward data from offline devices.

If you only need to validate event behavior locally, start with C/POSIX.

## What endpoint do SDKs use?

Official SDKs send compact chunk frames to:

```text
POST /capture
Content-Type: application/vnd.honch.chunk
X-Honch-Project-Key: <project_api_key>
X-Honch-Stream-Id: <stream_id>
```

## Should I use HTTPS?

Yes in production. Use `https://capture.honch.io` with certificate verification enabled. Local HTTP is only for intentional local Capture testing.

## Does the SDK flush in the background?

No unmanaged background upload thread is hidden from your application. Call the cooperative tick API from a scheduler or low-priority task, or call flush explicitly for an immediate drain attempt. On embedded targets, tick and flush can perform network I/O when uploads are enabled.

## What gets collected automatically?

Every event includes device identity, device model, firmware version, SDK platform, SDK version, and environment. Sessions add `$session_id`. Battery-aware SDKs can add `$battery_level` and `$battery_low`. Some embedded SDKs can add connectivity or Wi-Fi signal context where supported.

## Can user properties override SDK properties?

No. SDK-owned property keys are reserved and rejected before queueing. Use a different property name for custom values.

## What happens offline?

Events queue locally before upload. Retryable failures remain pending. Permanent rejections are dropped or moved to dead-letter storage depending on the SDK. ESP-IDF and Arduino ESP32 use volatile RAM queues by default; use caller-provided durable queue ops when pending events must survive reset or power loss.

## What does reset do?

`reset` is a factory-reset-style boundary. It clears queued data, clears session state, resets low-battery state, and restores or rotates identity depending on whether the device ID was explicitly configured.

## Why does MicroPython require a user C module?

The MicroPython package is a thin wrapper around `_honch_core`. Firmware must include the user C module; installing the Python wrapper alone is not enough.

## Can I call track from an ISR?

Do not call full SDK tracking APIs from ISR context. Tracking can validate input, take locks, allocate, encode, and queue. Move the signal into a normal task first.

## How does React Native Relay ACK a device?

The mobile relay ACKs after every chunk passes validation and the completed compact message is durably stored on mobile. Capture upload success is tracked separately by the relay queue and retry scheduler.
