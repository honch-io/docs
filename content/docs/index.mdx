---
title: Honch SDKs
description: Product analytics SDKs for firmware, embedded Linux, MicroPython, Arduino, and mobile relay flows.
---

Honch SDKs collect product analytics from connected hardware. They run inside firmware, embedded Linux software, MicroPython firmware, Arduino ESP32 sketches, or a mobile relay app, then queue device events locally and upload compact binary chunks to Honch Capture.

## Current SDKs

| SDK | Status | Use it for |
| --- | --- | --- |
| ESP-IDF | Available, component version `0.1.0` | ESP32-family products built with ESP-IDF. |
| C/POSIX | Available, core-derived `0.2.0` | Embedded Linux devices, local development harnesses, and C core validation. |
| MicroPython | Available, core-derived `0.2.0` | Firmware that can include the `_honch_core` MicroPython user C module. |
| Arduino ESP32 | In progress, library version `0.1.0` | ESP32 Arduino sketches that can tolerate the current milestone limitations. |
| React Native Relay | In progress, package version `0.1.0` | Companion apps that receive relay chunks from offline devices and upload them later. |

The portable C core is the source of truth for shared behavior: event semantics, identity, lifecycle events, queue policy, retry classification, compact wire-v2 encoding, and packetization. Platform ports provide storage, transport, timing, randomness, and platform-specific adapters.

## What Every SDK Sends

Each event has an event name, a timestamp, a `distinct_id`, and properties. SDK-owned properties such as `$device_id`, `$device_model`, `$firmware_version`, `$sdk_platform`, `$sdk_version`, and `$environment` are added automatically and win over user-supplied values with the same keys.

SDKs also emit lifecycle events where the platform supports them:

- `$device_boot`
- `$device_shutdown`
- `$firmware_update`
- `$battery_low`
- `$session_start`
- `$session_end`
- `$connectivity_change`

## Upload Contract

SDKs upload one compact chunk frame at a time:

```text
POST /capture
Content-Type: application/vnd.honch.chunk
X-Honch-Project-Key: <project_api_key>
X-Honch-Stream-Id: <stream_id>
```

Capture also accepts the same compact chunk format on `/e` and `/chunks`. Use the hosted HTTPS capture endpoint in production; local development endpoints can use HTTP when intentionally configured.
