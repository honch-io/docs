The goal of this page is one verified event. Pick your SDK, set five values, send a single event, and confirm it arrives. Deeper configuration lives on each SDK's guide.

## 1. Set Your Capture Values

Every integration needs the same handful of values:

| Value | Example | Notes |
| --- | --- | --- |
| Project API key | `your-project-key` | Sent as the `X-Honch-Project-Key` header. |
| Endpoint | `https://i.honch.io` | SDKs append `/capture`. Use HTTPS in production. |
| Device model | `demo-board` | Static per product. |
| Firmware version | `1.0.0` | Drives `$firmware_update` when it changes. |
| Environment | `production` | Optional; defaults to `production`. Use `development` for tests. |

## 2. Send A First Event

### ESP-IDF

```bash
idf.py add-dependency "honch/honch^0.2.4"
```

```c
#include "honch.h"
static uint8_t buf[16384];

honch_config_t config = {
    .api_key = "your-project-key",
    .host = "https://i.honch.io",
    .device_model = "demo-board",
    .firmware_version = "1.0.0",
    .event_buffer = buf,
    .event_buffer_size = sizeof(buf),
};
honch_init(&config);
honch_track("app_started", NULL, 0);
honch_flush();   // normally you call honch_tick() from a task
```

Full guide: [ESP-IDF](/sdks/esp-idf).

### C / POSIX

```bash
cmake -S . -B build -DHONCH_BUILD_EXAMPLES=ON && cmake --build build
```

```c
#include "honch/honch.h"

honch_config_t config = {
    .api_key = "your-project-key",
    .endpoint_url = "https://i.honch.io",
    .device_model = "linux-gateway",
    .firmware_version = "1.0.0",
    .queue_directory = "/var/lib/honch",
};
honch_client_t *client = NULL;
honch_init(&client, &config);
honch_track(client, "app_started", NULL, 0);
honch_flush(client);
```

Full guide: [C / POSIX](/sdks/c-posix).

### MicroPython

Build firmware with the `_honch_core` user C module (see the [guide](/sdks/micropython)), then:

```python
import honch
client = honch.Honch(
    api_key="your-project-key",
    endpoint_url="https://i.honch.io",
    device_id="dev-board-001",
    device_model="dev-board",
    firmware_version="1.0.0",
    event_buffer=bytearray(8192),
)
client.track("app_started")
client.flush()
```

Full guide: [MicroPython](/sdks/micropython).

### Arduino (ESP32)

```ini
lib_deps = honch/Honch@^0.2.4
```

```cpp
#include <Honch.h>
static uint8_t buf[8192];

HonchConfig config = {};
config.apiKey = "your-project-key";
config.host = "https://i.honch.io";
config.deviceModel = "demo-board";
config.firmwareVersion = "1.0.0";
config.rootCaPem = ROOT_CA_PEM;
config.eventBuffer = buf;
config.eventBufferSize = sizeof(buf);

honch::defaultClient().begin(config);
honch::defaultClient().track("app_started");
```

Full guide: [Arduino (ESP32)](/sdks/arduino).

### No SDK?

Send JSON straight to the API:

```bash
curl -X POST https://i.honch.io/capture \
  -H "Content-Type: application/json" \
  -H "X-Honch-Project-Key: your-project-key" \
  -d '{
    "context": {
      "distinct_id": "device-1",
      "$device_id": "device-1",
      "$device_model": "demo-board",
      "$firmware_version": "1.0.0",
      "$sdk_platform": "custom",
      "$sdk_version": "1.0.0"
    },
    "events": [{ "event": "app_started" }]
  }'
```

Full guide: [Build Your Own Integration](/byo-integration) and the [HTTP JSON API](/http-api).

## 3. Verify The First Event

| Check | What you should see |
| --- | --- |
| Init succeeds | The SDK returns success and queues `$device_boot`. |
| Flush attempts upload | A `POST /capture` goes out once connected. |
| Capture accepts | `204` (batch accepted) or `202` (chunk stored). JSON returns `200` with `accepted`. |
| Event appears | `app_started` shows up in your project's live events. |

If events queue but never upload, work through [Troubleshooting](/troubleshooting).

## 4. Add A Few Product Events

Once one event is verified, instrument the moments that matter. Good first events:

| Event | When |
| --- | --- |
| `app_started` | Firmware finished boot and is ready. |
| `mode_changed` | The device switched modes (property `mode`). |
| `sync_started` / `sync_finished` | A sync began/ended (property `duration_ms`). |
| `button_pressed` | A physical input fired (property `pin` or `button`). |

Keep names stable and put the variable detail in properties. Then read [Shared Concepts](/concepts) to understand identity, sessions, and how queueing and retries behave.
