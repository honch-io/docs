Pick the SDK that matches where your device can run code:

| Target | Start with |
| --- | --- |
| ESP32 firmware built with ESP-IDF | [ESP-IDF SDK](/sdks/esp-idf) |
| Embedded Linux, POSIX device process, or local C harness | [C/POSIX SDK](/sdks/c-posix) |
| MicroPython firmware with custom user modules | [MicroPython SDK](/sdks/micropython) |
| ESP32 Arduino sketches | [Arduino ESP32 SDK](/sdks/arduino) |
| BLE/offline device forwarding through a companion app | [React Native Relay](/sdks/react-native-relay) |

## ESP-IDF

Add the component from an ESP-IDF project:

```bash
idf.py add-dependency "honch-io/honch^0.1.0"
```

If you vendor manually, copy or submodule the whole SDK repository into `components/honch`; the ESP-IDF component includes shared files from the repository `core/` directory.

Initialize after NVS and Wi-Fi setup:

```c
#include "honch.h"
#include "nvs_flash.h"

static uint8_t event_buffer[16384];

void app_main(void)
{
    nvs_flash_init();
    // Initialize Wi-Fi and SNTP for your product.

    honch_config_t config = {
        .api_key = CONFIG_HONCH_API_KEY,
        .host = "https://capture.honch.io",
        .device_model = "demo-board",
        .firmware_version = "1.0.0",
        .event_buffer = event_buffer,
        .event_buffer_size = sizeof(event_buffer),
    };

    if (honch_init(&config) == HONCH_OK) {
        honch_track("app_started", "{\"source\":\"esp-idf\"}");
        honch_flush();
    }
}
```

Run the example:

```bash
cd ports/esp-idf/example
idf.py menuconfig
idf.py set-target esp32s3
idf.py flash monitor
```

## C/POSIX

Build the POSIX SDK from the SDK repository:

```bash
cd ports/posix
cmake -S . -B build -DHONCH_BUILD_TESTS=ON -DHONCH_BUILD_EXAMPLES=ON
cmake --build build
ctest --test-dir build --output-on-failure
```

Minimal use:

```c
#include "honch/honch.h"

int main(void)
{
    honch_config_t config = {
        .api_key = "project-key",
        .endpoint_url = "https://capture.honch.io",
        .device_model = "linux-gateway",
        .firmware_version = "1.0.0",
        .queue_directory = ".honch-queue",
        .durability_mode = HONCH_DURABILITY_SYNC_ALWAYS,
    };

    honch_client_t *client = NULL;
    if (honch_init(&client, &config) == HONCH_OK) {
        honch_track(client, "app_started", "{\"source\":\"posix\"}");
        honch_flush(client);
        honch_shutdown(client);
    }
}
```

## MicroPython

The MicroPython SDK is not pure Python. Build firmware with the `_honch_core` user C module, then freeze or install the thin `honch` wrapper:

```bash
make -C ports/unix \
  USER_C_MODULES=/path/to/SDK/ports/micropython/usermod/honch/micropython.cmake
```

Use the wrapper from MicroPython:

```python
import honch

client = honch.Honch(
    api_key="project-key",
    endpoint_url="https://capture.honch.io",
    device_model="pico-w",
    firmware_version="1.0.0",
    queue_directory="/honch",
)

client.track("app_started", {"source": "micropython"})
client.flush()
client.shutdown()
```

## Arduino ESP32

The Arduino SDK is an in-progress ESP32 wrapper. Include the library, connect Wi-Fi, and use the singleton:

```cpp
#include <WiFi.h>
#include <Honch.h>

static uint8_t eventBuffer[8192];

void setup() {
  WiFi.begin("ssid", "password");
  while (WiFi.status() != WL_CONNECTED) delay(250);

  HonchConfig config = {};
  config.apiKey = "project-key";
  config.host = "https://capture.honch.io";
  config.deviceModel = "esp32-devkit";
  config.firmwareVersion = "1.0.0";
  config.environment = "production";
  config.eventBuffer = eventBuffer;
  config.eventBufferSize = sizeof(eventBuffer);
  config.flushIntervalSeconds = 60;
  config.flushEventThreshold = 30;

  Honch.begin(config);
  Honch.track("app_started", "{\"source\":\"arduino\"}");
  Honch.flush();
}

void loop() {}
```

## React Native Relay

Use the relay when a firmware device cannot reach Capture directly and instead streams relay chunks over BLE to a companion app. The mobile app validates relay frames, stores completed compact messages durably, ACKs the device after durable receipt, and uploads the message to Capture.

```ts
import { NativeEventEmitter, NativeModules } from "react-native";
import { createMMKV } from "react-native-mmkv";
import {
  createMobileRelay,
  createMmkvRelayStore,
  createRelayNativeBindings,
} from "@honch/react-native-relay";

const nativeModule = NativeModules.HonchReactNativeRelay;
const bindings = createRelayNativeBindings(nativeModule);

const relay = createMobileRelay({
  durableStore: createMmkvRelayStore(createMMKV({ id: "honch-relay" })),
  uploaderConfig: {
    endpointUrl: "https://capture.honch.io",
    projectKey: "project-key",
    relayId: "mobile-relay-1",
    relaySdkPlatform: "react-native",
    relaySdkVersion: "0.1.0",
    streamId: (message) => `relay-${message.deviceId}`,
    messageId: (message) => Number(message.sequence),
  },
  bleNative: bindings.bleNative,
  schedulerNative: bindings.schedulerNative,
  frameEvents: new NativeEventEmitter(nativeModule),
});

relay.subscribeNativeFrames();
```

## Common Next Step

After the first event is queued, verify transport logs and queue behavior before shipping. All SDKs use the compact `POST /capture` contract and preserve retryable failures; permanent capture rejections are dropped or dead-lettered depending on the port.
