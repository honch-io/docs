This is the shortest safe path through Honch: choose the SDK your device can support, send one event, and verify local queue or delivery behavior before adding product tracking.

## 1. Choose An SDK

| Target | Use | Start here |
| --- | --- | --- |
| ESP32 firmware built with ESP-IDF | ESP-IDF component | [ESP-IDF guide](/sdks/esp-idf) |
| Embedded Linux, POSIX device process, or local C harness | C/POSIX SDK | [C/POSIX guide](/sdks/c-posix) |
| MicroPython firmware with custom user modules | MicroPython SDK plus `_honch_core` | [MicroPython guide](/sdks/micropython) |
| ESP32 Arduino sketches | Arduino ESP32 library | [Arduino guide](/sdks/arduino) |
| BLE/offline forwarding through a companion app | React Native Relay | [React Native Relay guide](/sdks/react-native-relay) |

If you are not sure, start with C/POSIX on a development machine. It exercises the same shared behavior as the embedded ports without flashing hardware.

## 2. Set Capture Values

Direct-upload SDKs need:

| Value | Example | Notes |
| --- | --- | --- |
| Project API key | `project-key` | Sent as `X-Honch-Project-Key`. |
| Capture endpoint | `https://capture.honch.io` | SDKs append `/capture`. |
| Device model | `demo-board` | Product or board model. |
| Firmware version | `1.0.0` | Used for filtering and firmware-update events. |
| Environment | `production` | Defaults to `production` when omitted. |

Production firmware should use the hosted HTTPS endpoint. Local HTTP is only for an intentional local Capture service.

## 3. Send One Event

### ESP-IDF

```bash
idf.py add-dependency "honch-io/honch^0.2.0"
```

```c
#include "honch.h"
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"

static uint8_t event_buffer[16384];

static void honch_telemetry_task(void *arg)
{
    (void)arg;
    for (;;) {
        honch_tick();
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}

void app_main(void)
{
    // Initialize Wi-Fi, time, and TLS trust before expecting delivery.

    honch_config_t config = {
        .api_key = CONFIG_HONCH_API_KEY,
        .host = "https://capture.honch.io",
        .device_model = "demo-board",
        .firmware_version = "1.0.0",
        .event_buffer = event_buffer,
        .event_buffer_size = sizeof(event_buffer),
    };

    if (honch_init(&config) == HONCH_OK) {
        const honch_property_t properties[] = {
            honch_prop("source", honch_str("esp-idf")),
        };
        honch_track("app_started", properties, 1);
        xTaskCreate(honch_telemetry_task, "honch_telemetry", 4096, NULL, 2, NULL);
    }
}
```

### C/POSIX

```bash
cd ports/posix
cmake -S . -B build -DHONCH_BUILD_EXAMPLES=ON
cmake --build build
```

```c
#include "honch/honch.h"

int main(void)
{
    honch_config_t config = {
        .api_key = "project-key",
        .endpoint_url = "https://capture.honch.io",
        .device_model = "linux-gateway",
        .firmware_version = "1.0.0",
        .queue_directory = ".honch-queue",
        .durability_mode = HONCH_DURABILITY_SYNC_ALWAYS,
    };

    honch_client_t *client = NULL;
    if (honch_init(&client, &config) == HONCH_OK) {
        const honch_property_t properties[] = {
            honch_prop("source", honch_str("posix")),
        };
        honch_track(client, "app_started", properties, 1);
        honch_flush(client);
        honch_shutdown(client);
    }
}
```

### MicroPython

Build firmware with `_honch_core`; installing Python files alone is not enough.

```bash
make -C ports/unix \
  USER_C_MODULES=/path/to/SDK/ports/micropython/usermod/honch/micropython.cmake
```

```python
import honch

client = honch.Honch(
    api_key="project-key",
    endpoint_url="https://capture.honch.io",
    device_id="device-serial-001",
    device_model="pico-w",
    firmware_version="1.0.0",
    event_buffer=bytearray(8192),
)

client.track("app_started", {"source": "micropython"})
client.flush()
client.shutdown()
```

### Arduino ESP32

Arduino ESP32 is preview. Use it for evaluation or controlled pilots until your product has passed its own hardware validation.

```cpp
#include <WiFi.h>
#include <Honch.h>

static uint8_t eventBuffer[8192];
static const char HONCH_ROOT_CA_PEM[] = R"EOF(
-----BEGIN CERTIFICATE-----
REPLACE_WITH_CAPTURE_ENDPOINT_ROOT_CA
-----END CERTIFICATE-----
)EOF";

void setup() {
  WiFi.begin("ssid", "password");
  while (WiFi.status() != WL_CONNECTED) delay(250);

  HonchConfig config = {};
  config.apiKey = "project-key";
  config.host = "https://capture.honch.io";
  config.rootCaPem = HONCH_ROOT_CA_PEM;
  config.deviceModel = "esp32-devkit";
  config.firmwareVersion = "1.0.0";
  config.environment = "production";
  config.eventBuffer = eventBuffer;
  config.eventBufferSize = sizeof(eventBuffer);
  config.flushMinIntervalMs = 10000;

  if (Honch.begin(config)) {
    const honch_property_t properties[] = {
      honch_prop("source", honch_str("arduino")),
    };
    Honch.track("app_started", properties, 1);
  }
}

void loop() {
  Honch.tick();
}
```

### React Native Relay

React Native Relay is preview and is only for devices that cannot upload directly. The mobile app ACKs the device after the completed relay message is durably stored, then uploads to Capture separately.

```ts
import { NativeEventEmitter, NativeModules } from "react-native";
import { createMMKV } from "react-native-mmkv";
import {
  createMobileRelay,
  createMmkvRelayStore,
  createRelayNativeBindings,
} from "@honch/react-native-relay";

const nativeModule = NativeModules.HonchReactNativeRelay;
const bindings = createRelayNativeBindings(nativeModule);

const relay = createMobileRelay({
  durableStore: createMmkvRelayStore(createMMKV({ id: "honch-relay" })),
  uploaderConfig: {
    endpointUrl: "https://capture.honch.io",
    projectKey: "project-key",
    relayId: "mobile-relay-1",
    relaySdkPlatform: "react-native",
    relaySdkVersion: "0.1.0",
    streamId: (message) => `relay-${message.deviceId}`,
    messageId: (message) => Number(message.sequence),
  },
  bleNative: bindings.bleNative,
  schedulerNative: bindings.schedulerNative,
  frameEvents: new NativeEventEmitter(nativeModule),
});

relay.subscribeNativeFrames();
```

## 4. Verify

Check these in order:

1. The SDK init call succeeds.
2. The event call returns success or does not raise.
3. Local queue state exists, or the SDK reports a RAM queue depth.
4. The configured endpoint is the base host, and the SDK appends `/capture`.
5. Capture accepts the upload or the SDK leaves retryable data pending.

If the first event does not appear, stop adding events and use [Troubleshooting](/troubleshooting).
